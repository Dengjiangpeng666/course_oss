# 毕业要求达成度统一计算平台 — 数据库详细设计

> **数据库**: PostgreSQL 15+（开发阶段可用 SQLite） | **ORM**: SQLAlchemy 2.0 Async

---

## 1. 表总览

共 **13 张业务表**（不含 Alembic 迁移记录表）。

```
┌────────────────────────────┐
│  用户与配置 (2 张)           │
│  users, global_config       │
├────────────────────────────┤
│  毕业要求 (2 张)             │
│  graduation_requirements    │
│  graduation_indicators      │
├────────────────────────────┤
│  课程与学生 (3 张)           │
│  courses, classes, students │
├────────────────────────────┤
│  权重矩阵 (2 张)             │
│  macro_support_matrix       │
│  internal_weight_matrix     │
├────────────────────────────┤
│  大纲与成绩 (3 张)           │
│  course_objectives          │
│  assessment_items           │
│  student_scores             │
├────────────────────────────┤
│  计算结果 (1 张)             │
│  calculation_results        │
└────────────────────────────┘
```

### 表关系图


## 2. 用户与配置表

### 2.1 users — 用户表

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | CHAR(36) | PK, NOT NULL | UUID，由应用层生成 |
| username | VARCHAR(50) | NOT NULL, UNIQUE | 登录用户名 |
| password_hash | VARCHAR(255) | NOT NULL | bcrypt 哈希后的密码 |
| role | VARCHAR(20) | NOT NULL, CHECK | admin / academic / director / teacher |
| display_name | VARCHAR(100) | NOT NULL | 用户显示姓名 |
| college | VARCHAR(100) | NULL | 所属学院 |
| major | VARCHAR(100) | NULL | 所属专业 |
| is_active | BOOLEAN | NOT NULL, DEFAULT TRUE | 账号启用状态 |
| created_at | TIMESTAMP | NOT NULL, DEFAULT NOW() | 创建时间 |
| updated_at | TIMESTAMP | NOT NULL, DEFAULT NOW() | 更新时间（自动更新） |

**索引**:
```sql
CREATE UNIQUE INDEX idx_users_username ON users(username);
CREATE INDEX idx_users_role ON users(role);
```

## 5. 权重矩阵表

### 5.1 macro_support_matrix — 宏观支撑矩阵（课程 ↔ 指标点，W）

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | CHAR(36) | PK, NOT NULL | UUID |
| course_id | CHAR(36) | NOT NULL, FK → courses.id, ON DELETE CASCADE | 课程 |
| indicator_id | CHAR(36) | NOT NULL, FK → graduation_indicators.id, ON DELETE CASCADE | 指标点 |
| weight_W | FLOAT | NOT NULL, CHECK (weight_W >= 0 AND weight_W <= 1) | 课程对该指标点的总支撑权重 |
| created_at | TIMESTAMP | NOT NULL, DEFAULT NOW() | |
| updated_at | TIMESTAMP | NOT NULL, DEFAULT NOW() | |

**索引**:
```sql
CREATE UNIQUE INDEX idx_macro_course_indicator ON macro_support_matrix(course_id, indicator_id);
CREATE INDEX idx_macro_course ON macro_support_matrix(course_id);
CREATE INDEX idx_macro_indicator ON macro_support_matrix(indicator_id);
```

**业务约束**（应用层校验，不在 DB 层面强制）:
- 对任意 indicator_id，其关联的所有 weight_W 之和 = 1.0

**示例数据**:
```sql
INSERT INTO macro_support_matrix (id, course_id, indicator_id, weight_W) VALUES
  ('msm01', 'c001', 'gi0301', 0.3),
  ('msm02', 'c001', 'gi0303', 0.5),
  ('msm03', 'c001', 'gi0304', 0.2);
-- 校验: 3-1 ΣW=0.3 (若不唯一支撑，需其他课程补充到 1.0)
```

---

### 5.2 internal_weight_matrix — 内部贡献权重矩阵（课程目标 ↔ 指标点，w）

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | CHAR(36) | PK, NOT NULL | UUID |
| objective_id | CHAR(36) | NOT NULL, FK → course_objectives.id, ON DELETE CASCADE | 课程目标 |
| indicator_id | CHAR(36) | NOT NULL, FK → graduation_indicators.id, ON DELETE CASCADE | 指标点 |
| weight_w | FLOAT | NOT NULL, CHECK (weight_w >= 0 AND weight_w <= 1) | 该目标对指标点的内部贡献权重 |
| created_at | TIMESTAMP | NOT NULL, DEFAULT NOW() | |
| updated_at | TIMESTAMP | NOT NULL, DEFAULT NOW() | |

**索引**:
```sql
CREATE UNIQUE INDEX idx_internal_obj_indicator ON internal_weight_matrix(objective_id, indicator_id);
CREATE INDEX idx_internal_objective ON internal_weight_matrix(objective_id);
CREATE INDEX idx_internal_indicator ON internal_weight_matrix(indicator_id);
```

**业务约束**（应用层校验）:
- 在同一门课范围内，对任意 indicator_id，其关联的所有 weight_w 之和 = 1.0

**示例数据**（数据结构课程）:
```sql
INSERT INTO internal_weight_matrix (id, objective_id, indicator_id, weight_w) VALUES
  ('iwm01', 'obj001', 'gi0301', 1.0),  -- 目标1-1 → 3-1, w=1.0
  ('iwm02', 'obj002', 'gi0303', 0.5),  -- 目标2-1 → 3-3, w=0.5
  ('iwm03', 'obj003', 'gi0303', 0.5),  -- 目标2-2 → 3-3, w=0.5
  ('iwm04', 'obj004', 'gi0304', 1.0);  -- 目标3-1 → 3-4, w=1.0
-- 校验 3-3: Σw = 0.5+0.5 = 1.0 ✅
```

### 6.2 assessment_items — 考核点

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | CHAR(36) | PK, NOT NULL | UUID |
| course_id | CHAR(36) | NOT NULL, FK → courses.id, ON DELETE CASCADE | 所属课程 |
| objective_id | CHAR(36) | NOT NULL, FK → course_objectives.id, ON DELETE CASCADE | 绑定的课程目标（唯一） |
| name | VARCHAR(200) | NOT NULL | 考核点名称，如 "期末卷第1题" |
| max_score | FLOAT | NOT NULL, CHECK (max_score > 0) | 该考核点满分 |
| created_at | TIMESTAMP | NOT NULL, DEFAULT NOW() | |
| updated_at | TIMESTAMP | NOT NULL, DEFAULT NOW() | |

**索引**:
```sql
CREATE INDEX idx_items_course ON assessment_items(course_id);
CREATE INDEX idx_items_objective ON assessment_items(objective_id);
```

**约束说明**:
- 每个考核点绑定到**唯一**的课程目标（需求规格 B-3）
- 同一个客观考试项目（如"期末考试"）若同时支撑多个课程目标，需拆分为多个 assessment_item 行，每行绑定不同 objective_id

**示例数据**:
```sql
INSERT INTO assessment_items (id, course_id, objective_id, name, max_score) VALUES
  ('item01', 'c001', 'obj001', '在线作业(目标1-1)',   100),
  ('item02', 'c001', 'obj001', '课堂测试(目标1-1)',   100),
  ('item03', 'c001', 'obj003', '实验报告(目标2-2)',   100),
  ('item04', 'c001', 'obj004', '实验报告(目标3-1)',   100),
  ('item05', 'c001', 'obj002', '期末考试(目标2-1)',   100),
  ('item06', 'c001', 'obj003', '期末考试(目标2-2)',   100);
```

> 注意："实验报告" 和 "期末考试" 各拆为两行，分别绑定到不同目标。在成绩导入 Excel 中，name 必须唯一以匹配列名，因此加上 "(目标X-X)" 后缀。

### 6.3 student_scores — 学生原始成绩

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | CHAR(36) | PK, NOT NULL | UUID |
| student_id | CHAR(36) | NOT NULL, FK → students.id, ON DELETE CASCADE | 学生 |
| assessment_item_id | CHAR(36) | NOT NULL, FK → assessment_items.id, ON DELETE CASCADE | 考核点 |
| score | FLOAT | NOT NULL | 实际得分 |
| created_at | TIMESTAMP | NOT NULL, DEFAULT NOW() | |
| updated_at | TIMESTAMP | NOT NULL, DEFAULT NOW() | |

**索引**:
```sql
CREATE UNIQUE INDEX idx_scores_student_item ON student_scores(student_id, assessment_item_id);
CREATE INDEX idx_scores_student ON student_scores(student_id);
CREATE INDEX idx_scores_item ON student_scores(assessment_item_id);
```

**约束**:
- (student_id, assessment_item_id) 唯一 —— 同一学生对同一考核点只允许一条成绩记录
- `score` 可以为任何非负数值，不与 max_score 自动比较（允许超满分的情形，由教师自行判断）

## 7. 计算结果表

### 7.1 calculation_results — 三级计算结果

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | CHAR(36) | PK, NOT NULL | UUID |
| course_id | CHAR(36) | NOT NULL, FK → courses.id | 课程（program-level 时填 "PROGRAM"） |
| class_id | CHAR(36) | NULL, FK → classes.id | 班级（program-level 时为空） |
| level | VARCHAR(20) | NOT NULL, CHECK | objective / course / program |
| indicator_id | CHAR(36) | NULL, FK → graduation_indicators.id | 指标点（objective-level 时可为空） |
| objective_id | CHAR(36) | NULL, FK → course_objectives.id | 课程目标（course/program-level 时为空） |
| achievement_value | FLOAT | NOT NULL | 达成度值 |
| locked | BOOLEAN | NOT NULL, DEFAULT FALSE | 是否锁定 |
| locked_at | TIMESTAMP | NULL | 锁定时间 |
| triggered_by | CHAR(36) | NULL, FK → users.id | 触发计算的用户 |
| created_at | TIMESTAMP | NOT NULL, DEFAULT NOW() | |
| updated_at | TIMESTAMP | NOT NULL, DEFAULT NOW() | |

**索引**:
```sql
CREATE INDEX idx_results_course ON calculation_results(course_id);
CREATE INDEX idx_results_class ON calculation_results(class_id);
CREATE INDEX idx_results_level ON calculation_results(level);
CREATE INDEX idx_results_locked ON calculation_results(locked);
CREATE INDEX idx_results_course_level ON calculation_results(course_id, level);
```

**字段使用规则**（三种 level 的字段填充模式）:

| level | course_id | class_id | indicator_id | objective_id | 含义 |
|-------|-----------|----------|:---:|:---:|------|
| objective | 实际课程ID | 实际班级ID | NULL | 实际目标ID | C̄j: 某班级某课程目标的平均达成度 |
| course | 实际课程ID | 实际班级ID | 实际指标点ID | NULL | Ek: 某课程对某指标点的达成度 |
| program | "PROGRAM" | NULL | 实际指标点ID | NULL | Gk: 全专业对某指标点的总达成度 |

**锁定规则**:
- objective-level: locked = false（中间结果，不锁定）
- course-level: locked = true（教师触发计算后锁定）
- program-level: locked = true（全局计算后锁定）

## 8. 完整 DDL

```sql

CREATE TABLE users (
    id            CHAR(36) PRIMARY KEY,

    college       VARCHAR(100),
    major         VARCHAR(100),
    is_active     BOOLEAN NOT NULL DEFAULT TRUE,

    updated_at    TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP

CREATE TABLE global_config (
    id          CHAR(36) PRIMARY KEY,
    key         VARCHAR(100) NOT NULL,
    value       TEXT         NOT NULL,
    category    VARCHAR(50)  NOT NULL,
    description VARCHAR(255),
    created_at  TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at  TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);
CREATE UNIQUE INDEX idx_gc_key ON global_config(key);
CREATE INDEX idx_gc_category ON global_config(category);

-- ============================================
-- 2. 毕业要求
-- ============================================
CREATE TABLE graduation_requirements (
    id          CHAR(36) PRIMARY KEY,
    code        VARCHAR(20)  NOT NULL,
    title       VARCHAR(100) NOT NULL,
    description TEXT,
    created_at  TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at  TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);
CREATE UNIQUE INDEX idx_gr_code ON graduation_requirements(code);

CREATE TABLE graduation_indicators (
    id             CHAR(36) PRIMARY KEY,
    requirement_id CHAR(36) NOT NULL REFERENCES graduation_requirements(id) ON DELETE CASCADE,
    code           VARCHAR(20) NOT NULL,
    description    TEXT        NOT NULL,
    created_at     TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at     TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_gi_requirement ON graduation_indicators(requirement_id);

-- ============================================
-- 3. 课程与学生
-- ============================================
CREATE TABLE courses (
    id            CHAR(36) PRIMARY KEY,
    code          VARCHAR(50)  NOT NULL,
    name          VARCHAR(200) NOT NULL,
    credit        FLOAT,
    hours_theory  INTEGER,
    hours_lab     INTEGER,
    college       VARCHAR(100),
    major         VARCHAR(100),
    academic_year VARCHAR(20),
    semester      VARCHAR(20),
    created_at    TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at    TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);
CREATE UNIQUE INDEX idx_courses_code ON courses(code);

CREATE TABLE classes (
    id            CHAR(36) PRIMARY KEY,
    course_id     CHAR(36)     NOT NULL REFERENCES courses(id) ON DELETE CASCADE,
    name          VARCHAR(100) NOT NULL,
    academic_year VARCHAR(20)  NOT NULL,
    semester      VARCHAR(20)  NOT NULL,
    created_at    TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at    TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_classes_course ON classes(course_id);

CREATE TABLE students (
    id         CHAR(36) PRIMARY KEY,
    student_id VARCHAR(50)  NOT NULL,
    name       VARCHAR(100) NOT NULL,
    class_id   CHAR(36)     NOT NULL REFERENCES classes(id) ON DELETE CASCADE,
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_students_student_id ON students(student_id);
CREATE INDEX idx_students_class ON students(class_id);

-- ============================================
-- 4. 权重矩阵
-- ============================================
CREATE TABLE macro_support_matrix (
    id           CHAR(36) PRIMARY KEY,
    course_id    CHAR(36) NOT NULL REFERENCES courses(id) ON DELETE CASCADE,
    indicator_id CHAR(36) NOT NULL REFERENCES graduation_indicators(id) ON DELETE CASCADE,
    weight_W     FLOAT    NOT NULL CHECK (weight_W >= 0 AND weight_W <= 1),
    created_at   TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at   TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(course_id, indicator_id)
);
CREATE INDEX idx_macro_course ON macro_support_matrix(course_id);
CREATE INDEX idx_macro_indicator ON macro_support_matrix(indicator_id);

CREATE TABLE internal_weight_matrix (
    id           CHAR(36) PRIMARY KEY,
    objective_id CHAR(36) NOT NULL REFERENCES course_objectives(id) ON DELETE CASCADE,
    indicator_id CHAR(36) NOT NULL REFERENCES graduation_indicators(id) ON DELETE CASCADE,
    weight_w     FLOAT    NOT NULL CHECK (weight_w >= 0 AND weight_w <= 1),
    created_at   TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at   TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(objective_id, indicator_id)
);
CREATE INDEX idx_internal_objective ON internal_weight_matrix(objective_id);
CREATE INDEX idx_internal_indicator ON internal_weight_matrix(indicator_id);

-- ============================================
-- 5. 大纲与成绩
-- ============================================
CREATE TABLE course_objectives (
    id          CHAR(36) PRIMARY KEY,
    course_id   CHAR(36)    NOT NULL REFERENCES courses(id) ON DELETE CASCADE,
    code        VARCHAR(20) NOT NULL,
    description TEXT        NOT NULL,
    dimension   VARCHAR(50) NOT NULL,
    created_at  TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at  TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_objectives_course ON course_objectives(course_id);

CREATE TABLE assessment_items (
    id           CHAR(36) PRIMARY KEY,
    course_id    CHAR(36)     NOT NULL REFERENCES courses(id) ON DELETE CASCADE,
    objective_id CHAR(36)     NOT NULL REFERENCES course_objectives(id) ON DELETE CASCADE,
    name         VARCHAR(200) NOT NULL,
    max_score    FLOAT        NOT NULL CHECK (max_score > 0),
    created_at   TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at   TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_items_course ON assessment_items(course_id);
CREATE INDEX idx_items_objective ON assessment_items(objective_id);

CREATE TABLE student_scores (
    id                 CHAR(36) PRIMARY KEY,
    student_id         CHAR(36) NOT NULL REFERENCES students(id) ON DELETE CASCADE,
    assessment_item_id CHAR(36) NOT NULL REFERENCES assessment_items(id) ON DELETE CASCADE,
    score              FLOAT    NOT NULL,
    created_at         TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at         TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(student_id, assessment_item_id)
);
CREATE INDEX idx_scores_student ON student_scores(student_id);
CREATE INDEX idx_scores_item ON student_scores(assessment_item_id);

-- ============================================
-- 6. 计算结果
-- ============================================
CREATE TABLE calculation_results (
    id                CHAR(36) PRIMARY KEY,
    course_id         CHAR(36) NOT NULL REFERENCES courses(id),
    class_id          CHAR(36)          REFERENCES classes(id),
    level             VARCHAR(20) NOT NULL CHECK (level IN ('objective','course','program')),
    indicator_id      CHAR(36)          REFERENCES graduation_indicators(id),
    objective_id      CHAR(36)          REFERENCES course_objectives(id),
    achievement_value FLOAT      NOT NULL,
    locked            BOOLEAN NOT NULL DEFAULT FALSE,
    locked_at         TIMESTAMP,
    triggered_by      CHAR(36)          REFERENCES users(id),
    created_at        TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at        TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_results_course ON calculation_results(course_id);
CREATE INDEX idx_results_class ON calculation_results(class_id);
CREATE INDEX idx_results_level ON calculation_results(level);
CREATE INDEX idx_results_locked ON calculation_results(locked);
CREATE INDEX idx_results_course_level ON calculation_results(course_id, level);
```

## 9. 索引策略说明

| 表 | 索引 | 类型 | 理由 |
|----|------|:---:|------|
| users | username | UNIQUE | 登录查询，高频 |
| users | role | 普通 | 按角色筛选用户列表 |
| global_config | key | UNIQUE | 配置键唯一性 |
| global_config | category | 普通 | 按类别批量获取下拉选项 |
| graduation_requirements | code | UNIQUE | 编号唯一 |
| graduation_indicators | requirement_id | 普通 | 按毕业要求查询下属指标点（联表） |
| courses | code | UNIQUE | 课程代码唯一 |
| classes | course_id | 普通 | 查询某课程的所有教学班 |
| students | student_id | 普通 | 成绩导入时按学号匹配 |
| students | class_id | 普通 | 查询某班级的所有学生 |
| macro_support_matrix | (course_id, indicator_id) | UNIQUE | 同一课程-指标点对唯一 |
| macro_support_matrix | course_id | 普通 | 查询某课程支撑的所有指标点 |
| macro_support_matrix | indicator_id | 普通 | 查询某指标点由哪些课程支撑（权重聚合查询） |
| internal_weight_matrix | (objective_id, indicator_id) | UNIQUE | 同一目标-指标点对唯一 |
| internal_weight_matrix | objective_id | 普通 | 查询某目标的所有指标点权重 |
| internal_weight_matrix | indicator_id | 普通 | 按指标点聚合权重（Σw=1.0 校验） |
| course_objectives | course_id | 普通 | 查询某课程的全部目标 |
| assessment_items | course_id | 普通 | 查询某课程的全部考核点 |
| assessment_items | objective_id | 普通 | 按目标查询考核点 |
| student_scores | (student_id, assessment_item_id) | UNIQUE | Upsert 定位 |
| student_scores | student_id | 普通 | 查询某学生的全部成绩 |
| student_scores | assessment_item_id | 普通 | 查询某考核点的全部成绩 |
| calculation_results | course_id | 普通 | 查询某课程的计算结果 |
| calculation_results | (course_id, level) | 复合 | 进度看板：查某课程是否有 course-level locked 记录 |
| calculation_results | locked | 普通 | 全局计算：查所有 locked 的 course-level 记录 |

## 10. 外键级联策略

| 父表 → 子表 | 级联策略 | 含义 |
|-------------|:---:|------|
| graduation_requirements → graduation_indicators | CASCADE | 删除毕业要求时，自动删除下属指标点 |
| graduation_indicators → macro_support_matrix | CASCADE | 删除指标点时，清除相关宏观权重 |
| graduation_indicators → internal_weight_matrix | CASCADE | 删除指标点时，清除相关内部权重 |
| courses → classes | CASCADE | 删除课程时，自动删除所有教学班 |
| courses → course_objectives | CASCADE | 删除课程时，自动删除所有目标 |
| courses → assessment_items | CASCADE | 删除课程时，自动删除所有考核点 |
| courses → macro_support_matrix | CASCADE | 删除课程时，清除其宏观权重 |
| course_objectives → internal_weight_matrix | CASCADE | 删除目标时，清除其内部权重 |
| course_objectives → assessment_items | CASCADE | 删除目标时，自动删除其下的考核点 |
| classes → students | CASCADE | 删除班级时，自动删除学生名单 |
| assessment_items → student_scores | CASCADE | 删除考核点时，自动删除相关成绩 |
| students → student_scores | CASCADE | 删除学生时，自动删除其所有成绩 |
| graduation_indicators → calculation_results | NO ACTION | 指标点被删时保留历史计算结果 |
| courses → calculation_results | NO ACTION | 课程被删时保留历史计算结果 |
| users → calculation_results (triggered_by) | SET NULL | 用户被删时，计算结果保留但触发者置空 |
